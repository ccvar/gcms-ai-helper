CCVAR 撰稿助手 · 使用帮助

一句话：这个软件让 AI 替你给网站写内容 → 审核 → 发布，支持文章、链接、页面三类内容，可以接到一个或多个你自己的网站。你只要点几下，或者干脆托管成全自动。默认只生成草稿、绝不偷偷发布——发布永远由你点头（除非你自己开了全自动）。

一、第一次用（3 步上手）

菜单栏图标在屏幕右上角（一个小图标）。点它展开菜单。

① 配置 AI 引擎：菜单 → 全局设置… → 看底部「AI 引擎」。把 Claude、GPT 至少一个点「登录…」按钮（图标）授权登录（详见第十节）。两盏灯变绿就是配好了。
② 添加你的网站：菜单 → 站点管理… → 填【名称 / API 域名 / API 密钥】→「添加站点」。域名形如 https://你的站.com/api/admin/v1；密钥需含 publish 权限才能发布。
③ 定位 + 选题：加站后会问「要不要校准网站定位」——站上已有发布文章就选「AI 校准」（自动读文章填好定位）；全新空站选「手动填写」，照模板示例填（见第四节）。

之后就会每天自动写一篇文章草稿（默认手动模式：写好等你发）。也可随时点 立刻撰稿一篇。

二、三大能力总览

同一个网站，AI 能帮你产三类内容，各走各的、各自配置、互不干扰：

能力是什么怎么触发去哪审/发
文章博客长文（有选题、可带代码示例）每天自动写一篇，或点「立刻撰稿一篇」「待审草稿」
链接外链收录（你给网址，AI 写标题+介绍+分类）按需：本站设置→链接→「生成」「待审链接」
页面固定页面（关于、服务说明…）按需：本站设置→页面→「起草这些页面」「待审页面」

只有「文章」有每日定时；链接/页面是你点按钮才生成（低频、按需）。

三、菜单各项

菜单项作用
每日自动撰稿：运行中/已暂停顶部状态。下面一行是「上次/正在撰稿」的实时状态。
站点：xxx当前活动站。多站时点开可切换。
立刻撰稿一篇不等定时，马上写一篇文章。撰稿中右侧有旋转图标，菜单栏图标也会「呼吸」。
待审草稿 / 待审页面 / 待审链接等你处理的草稿。点开每条有预览（浏览器看效果，不发布）和确认发布上线。
待发布（自动）仅半自动/全自动出现：过审、排队等到点自动发。点一条=取消。
最近发布最近已上线的内容，点开在网站上查看。
编辑选题…打开选题库 topics.md，决定 AI 写什么文章（第四节，最重要）。
校准网站定位…让 AI 读站内已发布文章，自动校准「网站定位」并补几个新选题。
本站设置… / 全局设置…两类设置，分开（第九节）。
站点管理…添加/移除站点、改密钥（后面括号是站点数量）。
最近运行日志…一个实时刷新的窗口，看每次撰稿/审核/发布发生了什么（排错必看）。

待审草稿的图标含义（彩色图标，非表情）

  • 绿色印章勾：主编建议发布，后面带评分（如「·8分」）。
  • 橙色三角叹号：主编觉得需要你看一眼再定。
  • 红色八角叉：主编不建议发。
  • 灰色文档：还没审（手动模式不会自动审）。

子菜单顶部有「图例」对照同样的图标。标题太长会截断成「…」，鼠标悬停看完整标题和主编意见。条目很多时只列最近若干条，点「打开完整清单」看全部。

四、文章：选题怎么定 ⭐

这是你最该掌握的。点 编辑选题… 打开 topics.md，里面三块：

区块作用你怎么用
网站定位告诉 AI「这网站是干嘛的、给谁看、什么调性」。用一段话描述。AI 据此把握风格、补题。
写作方向
(主题范围 = 边界)		允许的题材范围。AI 只在这些方向里自动补题。列几条方向，每行一个。删一条，AI 就不写那类了。
待写队列你指定要写的具体题目，从上往下取。想写什么直接写一行。队列空了，AI 才按「写作方向」自己拟题。
已写历史记录（自动追加），用来防撞题。不用管，自动维护。

三种玩法都支持，取决于你填哪块：

  • 给具体题目 → 写进「待写队列」，AI 照着写。
  • 给范围让它发挥 → 只填「写作方向」、清空「待写队列」，AI 在范围内自己想题。
  • 只说网站是干嘛的 → 写好「网站定位」+「写作方向」，其余交给它。

偷懒法（推荐）：点 「校准网站定位…」，AI 读你站里已发布的文章，自动把「网站定位/写作方向」写准、并补几个没写过的新选题。内容多了随时再点一次重新校准。

五、页面：批量起草（关于、服务说明…）

页面是固定、少量的门面内容，不该天天自动生成。做法：

  1. 「本站设置」→ 页面 tab → 在框里每行写一个：标题 | 这页讲什么（例：关于我们 | 团队、使命、联系方式）。
  2. 点 「起草这些页面」 → AI 逐个写草稿（每个约一两分钟，菜单状态行可看进度）。
  3. 去菜单 「待审页面」 → 预览 → 确认发布。

页面建议用「无计划」模式（纯手动·你审了再发，门面要把关）。生成成功的标题会在框里标成「# 已起草…」，不会重复生成；想重做就去掉前面的 #。

六、链接：贴网址，AI 补全

把外部网址收进你的「链接库」，AI 自动写好标题/介绍/分类：

  1. 「本站设置」→ 链接 tab → 每行贴一个网址（可选 网址 | 提示）。
  2. 勾选「增强」：自动归类（AI 选分类）、自动配图（生成封面，更费额度）。可设「介绍字数」。
  3. 点 「生成」 → 脚本先抓取每个网页的标题/摘要当素材，AI 据此写标题 + 详细介绍 + 一句话摘要，并选分类，建为草稿。
  4. 去菜单 「待审链接」 → 预览 → 发布（链接低风险，也可在链接 tab 设成全自动=生成即发）。

AI 不会编造网址——网址是你提供的真实地址，它只负责写描述、选分类。生成成功的网址同样会标成「# 已收录…」，不重复。

七、发布模式 / 否决窗口

每类内容、每个站各自设发布模式（在「本站设置」对应 tab）：

模式含义
手动（文章）/ 无计划（页面·链接）只生成草稿，等你手动点发布。最稳，默认。
半自动生成（文章还会主编审核给建议）后，进「待发布」等"否决窗口"小时数，到点自动发；这段时间你可在「待发布」取消。
全自动文章=过审到点自动发；链接=生成即发。适合托管。
  • 否决窗口：仅半自动/全自动用——过审后等几小时才真发，给你反悔时间。设 0 = 立即发。
  • 每日上限：全局设置里，全自动每天最多发几篇（跨站合计），防失控。
  • 页面没接自动发布，永远进「待审页面」等你手动发（门面需审）。

八、待审 / 预览 / 发布

  • 预览：点击后弹「生成中」小窗，好了自动在浏览器打开。优先用网站的在线预览（真站主题），不可用时回退本地排版页。预览不发布。
  • 确认发布上线：会二次确认。发布是公开动作——需要你的密钥含 publish 权限。
  • 待发布（自动）：半/全自动排队中的，点一条即取消。

九、设置：全局 vs 本站

设置拆成两个窗口，别混：

全局设置（所有站点共享）本站设置（只作用当前站）
每日撰稿时间
写作/审核引擎+型号（可混搭：GPT 写 + Claude 审）
Claude 令牌 / OpenAI Key
AI 引擎登录状态
AI 命令（高级逃生口，一般留空）		三个 tab：文章 / 链接 / 页面
各自：发布模式、目标字数、增强（归类/配图/含代码示例）
文章还有：否决窗口、写作/译文语种
链接/页面：清单 + 生成按钮

密钥不在「本站设置」里改了——去「站点管理」每个站后面的「改密钥」。

十、AI 引擎：登录 / 安装 / 状态灯

「全局设置」底部「AI 引擎」一行有 Claude、GPT 两个引擎，右侧是图标按钮（鼠标悬停看文字）：

  • 绿灯=已就绪。按钮变「切换账号」。
  • 灰灯=装了但没登录。点「登录」（钥匙图标）→ 自动开终端走浏览器授权。
    • GPT：codex login 用 ChatGPT 登录（有免费额度），登录完点「重新检测」即可，无需复制 key。
    • Claude：claude setup-token 会在终端打印一串令牌，复制粘到上面「Claude 令牌」框（需 Claude 订阅账号）。
  • 橙灯=没装命令。点「安装」（下载图标）看引导（用 npm install -g 装，需先有 Node.js）。

配好哪个引擎就能用哪个。也可两个都配，写作/审核分别选不同引擎。

十一、内容参数 / 语种

  • 目标字数（文章/页面）/ 介绍字数（链接）：填了 AI 按这个长度写；留空=用默认（文章 1500-2200、页面 500-1200、链接 2-4 句）。
  • 含代码示例（文章）：默认开；非技术站可关掉。
  • 自动归类：让 AI 选最贴切的目录/分类。
  • 自动配图：生成品牌封面图（更费额度，默认关）。
  • 写作语种：写哪种语言（站点支持的语种自动列出）。
  • 译文方式 / 译文语种：要不要额外产其它语种译文。不勾任何译文语种 = 只写写作语种、不翻译（此时「译文方式」自动置灰）。

十二、多站点

  • 各站平级：每个站独立的密钥、选题、发布策略、三类内容队列；没有"主站"之分。
  • 切换：菜单「站点：xxx」点开选另一个站。切换后菜单内容随之变成那个站的。
  • 添加/移除：「站点管理」里加站、改密钥、移除（移除只是取消注册，数据文件保留；至少留一个站）。
  • 全局 vs 每站：引擎/登录/时间是全局共享；密钥/选题/发布策略/字数是每站各自的。

十三、安全底线

  • 默认只生成草稿，发布永远要你确认（或你主动开半/全自动）。
  • 密钥/令牌存在本地、权限 600、不进 Git，不外泄。
  • 半/全自动有「否决窗口」+「每日上限」兜底。
  • AI 被严格限制：只通过本工具操作你的站、只碰内容工作文件、绝不动系统或其它文件。

十四、换电脑 / 发给别人

换自己的电脑：拷整个文件夹（含密钥、配置、选题、脚本），在新机跑一次 bash automation/install.sh（重建定时器/自启/App），再各自登录 AI（claude setup-token / codex login）。

发给别人：跑 bash automation/make-template.sh 会在 dist/ 生成一份零站点、无密钥的干净包（含 App + 快速开始.txt）。对方解压、配好 AI 与自己的站点即可用——不会带上你的任何数据。

十五、常见问题排错

撰稿一直"处理中"/没产出多半是引擎没登录。看「全局设置」灯是否绿；或菜单「最近运行日志」看报错。
发布失败密钥没有 publish 权限。去「站点管理→改密钥」换一个有发布权限的。
菜单显示"还没有任何站点"正常——还没加站。点「添加站点」即可。
设置窗口按钮点不动菜单栏应用偶发；本版已修。若仍遇到，关掉窗口重开。
状态行文字被截断鼠标悬停看全文；它会自动省略号，不会跑出菜单。
"否决窗口=0"还有待发布已修：veto=0 时过审即发，不再排队。
预览打不开/很慢会先弹"生成中"小窗；网络慢请稍候，好了自动打开。在线不可用会自动回退本地页。

还有问题：菜单「最近运行日志…」是排错第一现场，撰稿/审核/发布的每一步都在里面。
